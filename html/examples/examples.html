Manual de Formato

Este capítulo recorre todas las construcciones sintácticas disponibles en el sistema. Cada sección muestra primero la sintaxis fuente y luego el resultado renderizado.

1. Bloques de Código

Los bloques de código se delimitan con tres backticks y un identificador de lenguaje estándar. Para mostrar el código fuente sin highlight se deja el bloque sin lenguaje.

1.1 JavaScript

Sintaxis:

```js
function suma(a, b) {
  return a + b;
}
console.log(suma(2, 3));
```

Resultado:

function suma(a, b) {
  return a + b;
}
console.log(suma(2, 3));

1.2 Python

print(suma(2, 3))

Resultado:

print(suma(2, 3))

5. Citas y Epígrafos

Las citas se escriben como blockquote estándar precedido por un comentario <!-- quote: {…} --> que porta los metadatos (autor, título, opciones).

5.1 Cita con atribución

Una cita con autor y título de obra:

<!-- quote: {author: "George Orwell", title: "1984"} -->

> La lengua corrupta corrompía el pensamiento.

Resultado:

La lengua corrupta corrompía el pensamiento.

George Orwell, 1984

5.2 Epigrafo de capítulo

Un epigrafo aparece antes del título en la versión PDF. Se añade chapter: true en el comentario:

<!-- quote: {author: "Alan Turing", title: "Computing Machinery and Intelligence", chapter: true} -->

> We may only have a short time with each other,
> but it will take eternity to forget you.

Resultado:

We may only have a short time with each other, but it will take eternity to forget you.

Alan Turing, Computing Machinery and Intelligence

6. Hints (Ejercicios)

Los bloques de pista se renderizas como desplegables en HTML interactivo y se eliminan en PDF. Se marcan con comentarios <!-- hint --> y <!-- /hint -->:

<!-- hint -->

La clave está en usar un bucle `for...of` en lugar de un índice numérico.

<!-- /hint -->

Resultado:

Display hints...

La clave está en usar un bucle for...of en lugar de un índice numérico.

7. Términos de Índice

Los términos de índice solo tienen efecto en la compilación PDF. En HTML y EPUB se ignoran.

7.1 Término inline

Un término inline se marca con comentarios (( y )) alrededor de la palabra:

El concepto de <!-- index-inline -->recursión<!-- /index-inline --> es fundamental en programación.

Resultado:

El concepto de recursión es fundamental en programación.

7.2 Directivas de bloque

Se pueden añadir entradas al índice sin texto visible mediante comentarios:

<!-- index: "estructura de datos" -->
<!-- index: ["árbol", "árbol binario"] -->
<!-- indexsee: "hash", "tabla hash" -->

Estas directivas no producen texto visible pero generan entradas en el índice del PDF.

9. Nombres de Teclas

Los nombres de teclas se escriben con la etiqueta HTML <kbd>, que GitHub renderiza con estilo de tecla física. El script los convierte a [texto]{keyname}.

Guarda el archivo con <kbd>Ctrl+S</kbd> o abre el menú con <kbd>Alt+F</kbd>.

Resultado:

Guarda el archivo con Ctrl+S o abre el menú con Alt+F.

Otros ejemplos: Enter, Escape, Tab.

10. Sub y Superíndices

Se usan las etiquetas HTML <sub> y <sup> que GitHub renderiza directamente. El script las convierte a ~…~ y ^…^.

10.1 Subíndice

H<sub>2</sub>O es la fórmula del agua.

Resultado:

H2O es la fórmula del agua.

10.2 Superíndice

La complejidad es O(n<sup>2</sup>) en el peor caso.

Resultado:

La complejidad es O(n2) en el peor caso.

10.3 Combinado

Se puede combinar sub y superíndice en la misma expresión:

La fórmula general es x<sub>i</sub><sup>2</sup> + y<sub>j</sub><sup>3</sup>.

Resultado:

La fórmula general es xi2 + yj3.

11. enlaces Internos

Los enlaces internos se escriben con la sintaxis estándar de Markdown. Si tienes un archivo 01_valores.md, puedes enlazarlo así:

[el capítulo de valores](valores)

El sistema busca el archivo que termine en _valores.md y genera el enlace correcto (.html en HTML, .xhtml en EPUB).

12. Bloques Ocultos e IDs

12.1 Bloques de código ocultos

La opción hidden: true oculta un bloque de código en el output. Útil para setup previo a un ejercicio. Se indica con un comentario <!-- code-options: {…} --> antes del bloque:

<!-- code-options: {hidden: true} -->

```js
// Este bloque no se ve al renderizar
let estado = { valor: 0 };
```

12.2 IDs manuales

La directiva de ID se indica con un comentario <!-- id: "identificador" --> inmediatamente antes del heading:

<!-- id: "seccion-importante" -->

## Esta sección tiene un id custom

Esto permite crear enlaces precisos hacia esa sección desde otro lugar:

[ir a la sección importante](#seccion-importante)

13. Elementos Estándar de Markdown

Esta sección cubre los elementos de formato básicos de GitHub Flavored Markdown. El script convert_gfm.mjs los pasa sin modificar al renderizador markdown-it, que los convierte a HTML/LaTeX nativamente. El objetivo es verificar que todos estos formatos se conservan correctamente a través de la cadena de conversión.

13.1 Negrita

Se delimita con doble asterisco **…** o doble guión bajo __…__:

Este texto tiene palabras en **negrita** y otras en **también negrita**.

Resultado:

Este texto tiene palabras en negrita y otras en también negrita.

13.2 Cursiva (itálica)

Se delimita con un solo asterisco *…* o un solo guión bajo _…_:

Este texto tiene palabras en _cursiva_ y otras en _también cursiva_.

Resultado:

Este texto tiene palabras en cursiva y otras en también cursiva.

13.3 Negrita y cursiva combinadas

Se combinan tres asteriscos ***…***:

Este texto es **_negrita y cursiva al mismo tiempo_**.

Resultado:

Este texto es negrita y cursiva al mismo tiempo.

13.4 Tachado (strikethrough)

Se delimita con doble tilde ~~…~~ (extensión GFM):

Esta palabra está ~~tachada~~ en el texto.

Resultado:

Esta palabra está tachada en el texto.

13.5 Listas no ordenadas

Se crean con -, * o + al inicio de cada línea:

- Elemento uno
- Elemento dos
- Elemento tres

Resultado:

  • Elemento uno

  • Elemento dos

  • Elemento tres

13.6 Listas ordenadas (numeradas)

Se crean con un número seguido de punto N.:

1. Primer paso
2. Segundo paso
3. Tercer paso

Resultado:

  1. Primer paso

  2. Segundo paso

  3. Tercer paso

13.7 Listas anidadas

Se crean indentando los elementos hijos con 2-4 espacios. Se puede mezclar listas ordenadas y no ordenadas:

- Categoría A
  - Subcategoría A1
  - Subcategoría A2
- Categoría B
  1. Paso B1
  2. Paso B2

Resultado:

  • Categoría A

    • Subcategoría A1

    • Subcategoría A2

  • Categoría B

    1. Paso B1

    2. Paso B2

13.8 Blockquote estándar

Un blockquote sencillo se crea con > al inicio de cada línea. Se diferencia de las admoniciones (sección 4) en que no tiene [!TIPO] y de las citas con atribución (sección 5) en que no tiene <!-- quote -->:

> Esta es una cita estándar sin atribución.
> Puede ocupar varias líneas consecutivas.

Resultado:

Esta es una cita estándar sin atribución. Puede ocupar varias líneas consecutivas.

13.9 Regla horizontal

Se crea con tres o más guiones ---, asteriscos *** o guiones bajos ___. Debe estar separada por líneas en blanco para no interpretarse como subheading (setext):

Párrafo antes de la regla.

---

Párrafo después de la regla.

Resultado:

Párrafo antes de la regla.

Párrafo después de la regla.

13.10 Código inline con backticks

Se encierra entre un par de backticks `…`:

Para declarar una variable se usa `const` o `let` en JavaScript.

Resultado:

Para declarar una variable se usa const o let en JavaScript.

13.11 Bloque de código indentado

Cuatro espacios o un tab al inicio de cada línea producen un bloque de código sin highlight, equivalente a un bloque fenced sin identificador de lenguaje:

    function hola() {
        return "Hola mundo";
    }

Resultado:

function hola() {
    return "Hola mundo";
}

Nota: Una indentación menor de 4 espacios (1-3 espacios) no se conserva en el output renderizado: Markdown la colapsa y trata la línea como párrafo normal. Para márgenes visuales se recomienda usar blockquote (>) o un bloque de código.

El proyecto se documenta en [GitHub](https://github.com).

Resultado:

El proyecto se documenta en GitHub.
